# ТЗ: сбор яиц за несколько дней и режим замеса

Статус: реализовано 2026-07-11, базовый коммит реализации `20df4eb`. Документ оставлен как техническая спецификация и критерии приемки для регрессионной проверки.

## Цель

Добавлен учет сбора яиц за несколько дней и исправлен сценарий замеса смеси так, чтобы режим "уже скормленный" выбирался до чек-листа ингредиентов.

## Опорная техническая информация

- Telegram-логика яиц находится в `app/handlers/eggs.py`.
- Клавиатуры яиц находятся в `app/keyboards/eggs.py`.
- Бизнес-логика яиц находится в `app/services/eggs.py`.
- Хранилище яиц находится в `app/storage/repositories/eggs.py`.
- Таблица `egg_entries` уже хранит несколько записей на одну дату.
- Статистика яиц считает суммы по датам через `SUM(eggs_count)` и `GROUP BY entry_date`.
- Поэтому сбор за несколько дней не требует новой таблицы: нужно создать несколько обычных `egg_entries`, по одной записи на дату.
- Telegram-логика склада и замесов находится в `app/handlers/feeds.py`.
- Клавиатуры склада находятся в `app/keyboards/feeds.py`.
- Исправленный баг смеси был связан с тем, что после ввода количества замесов пользователь попадал в чек-лист, а кнопка "Записать как уже скормленный" находилась внутри чек-листа.

## Яйца: пользовательский сценарий

В разделе `Яйца -> Добавить яйца` первым шагом бот спрашивает тип операции:

```text
Какой сбор записать?

[Обычный сбор]
[Сбор за несколько дней]
```

`Обычный сбор` сохраняет текущую логику: выбор даты и ввод количества.

`Сбор за несколько дней` запускает отдельный поток:

```text
Введите общее количество собранных яиц
-> Как определить период?
   [Указать количество дней]
   [Не помню период]
```

## Ручной период

Если пользователь выбирает `Указать количество дней`, бот спрашивает:

```text
За сколько дней распределить сбор?
```

Значение должно быть целым числом от `2` до `30`.

Распределение должно работать для любого положительного количества яиц:

```text
base = total_eggs // days
remainder = total_eggs % days
```

Остаток добавляется к самым свежим датам.

Примеры:

- `15 / 3` -> `5, 5, 5`
- `20 / 3` -> `7, 7, 6`
- `40 / 3` -> `14, 13, 13`

## Неизвестный период

Если пользователь выбирает `Не помню период`, бот должен:

1. Рассчитать среднее количество яиц по дням с записями.
2. Если статистики нет или среднее `<= 0`, попросить указать количество дней вручную.
3. Рассчитать период как `ceil(total_eggs / average_eggs_per_recorded_day)`.
4. Ограничить период диапазоном `2..30`.
5. Найти ближайшие пустые дни назад от текущей даты.
6. Распределить яйца по найденным пустым дням.

Пустой день - дата, по которой у пользователя нет записей в `egg_entries`.

## Подтверждение

Перед записью бот должен показать предпросмотр:

```text
Распределить 20 яиц за 3 дня:

2026-07-11: 7 шт.
2026-07-10: 7 шт.
2026-07-09: 6 шт.

[Записать]
[Отмена]
```

Запись создается только после подтверждения.

## Хранение

Для каждой даты создается обычная запись `egg_entries`.

Комментарий:

- ручной период: `Сбор за несколько дней: N шт. за D дн.`
- неизвестный период: `Сбор за несколько дней: N шт. за D дн., период рассчитан автоматически`

## Смеси: исправление сценария

После выбора основы и ввода количества замесов бот показывает выбор режима:

```text
Как записать замес?

[Сделать сейчас]
[Записать как уже скормленный]
```

`Сделать сейчас` открывает текущий чек-лист.

`Записать как уже скормленный` сразу открывает выбор даты:

```text
Когда замес был сделан?

[Сегодня]
[7 дней назад]
[Без даты / не помню]
[Ввести дату]
```

Чек-лист не должен открываться в сценарии "уже скормленный".

## Критерии приемки

- Обычный сбор яиц работает как раньше.
- Сбор за несколько дней с ручным периодом создает несколько записей.
- Сбор за несколько дней с неизвестным периодом использует среднее и ближайшие пустые дни.
- Если статистики нет, авто-режим просит ввести период вручную.
- Перед сохранением есть предпросмотр.
- После сохранения история и статистика используют распределенные записи.
- После ввода количества замесов чек-лист не открывается сразу.
- Чек-лист открывается только после выбора `Сделать сейчас`.
- `Записать как уже скормленный` сразу ведет к выбору даты.

## Обязательные тесты

- Распределение работает для разных количеств: `15`, `20`, `40`.
- `20 / 3` -> `7, 7, 6`.
- `40 / 3` -> `14, 13, 13`.
- Неизвестный период считает дни по `ceil(total / average)`.
- При среднем `7`: `15 -> 3`, `20 -> 3`, `40 -> 6`.
- Авто-режим использует пустые дни.
- Обычный сбор яиц не сломан.
- После количества смеси показывается выбор режима, а не чек-лист.
- Режим "уже скормленный" ведет сразу к выбору даты.
- Полный `pytest` проходит.
