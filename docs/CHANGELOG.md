# Changelog

## 2026-05-26

- Исправлен fallback погоды: `wttr.in` теперь вызывается по координатам, чтобы кириллический город не ломал URL.
- Для погоды добавлен fallback на `wttr.in`, если Open-Meteo не отвечает.
- Источник погоды теперь показывается из сохраненной записи, а не жестко как Open-Meteo.
- Раздел `Яйца` больше не обновляет погоду автоматически при открытии, чтобы не подвешивать бота при недоступности Open-Meteo.
- На главной странице `Яйца` добавлен краткий блок погоды из сохраненных данных.
- Обновление погоды вынесено в ручную кнопку и выполняется вне основного event loop бота.
- Подключена погода к разделу `Яйца`: город геокодируется через Open-Meteo, дневная погода сохраняется в `daily_weather`.
- В расчетах яиц добавлена ориентировочная погодная поправка к недельному прогнозу.
- Экран `Город и погода` теперь показывает температуру, осадки, состояние погоды и кнопку обновления.
- Главное меню актуализировано под три рабочих раздела: `Корма`, `Инкубация`, `Яйца`.
- Добавлен раздел `Яйца`: ежедневный сбор, история за 14 дней, расчеты за 7/30 дней и прогноз на неделю.
- В расчетах яиц учитываются только взрослые несушки; цыплята и петухи исключены.
- Добавлены временные исключения несушек из яйценоскости: наседка, уход за цыплятами, линька, болезнь, другая причина.
- Добавлена настройка города для будущей погодной привязки яйценоскости; таблица `daily_weather` подготовлена, погодный API пока не подключен.
- Разделы и уведомления в настройках русифицированы и перенесены глубже в отдельный экран.
- Добавлен экран `О боте`: версия, GitHub, changelog, последний запуск, последний деплой и commit.
- Обычные деплои больше не отправляют шумное сообщение пользователям. Release notice отправляется только для `medium`, `major` и `critical`.
- Корма разделены по UX на `Поголовье` и `Стада`; стадо состоит из групп поголовья и ест назначенную готовую смесь.
- В стадах убрана лишняя кнопка `Поголовье`; переходы назад приведены к родительским разделам.
- Замес смеси переведен в подтверждаемый сценарий: сначала расчет количества, затем кнопка `Создать замес`.
- Расчеты по стадам учитывают готовую смесь на складе и потенциальные замесы из ингредиентов, показывают даты закупок.
- Складовой расчет получил сопоставление названий ингредиентов: дробленая кукуруза, зерносмесь вместо пшеницы, коммерческий комбикорм по названию.
- Обновлены тесты для кормов, меню, настроек, складовых расчетов и яиц.

## 2026-05-25

- Добавлен склад кормов: покупки, расчетные остатки, замесы смеси, история операций, фактическая корректировка и рационы поголовья.
- Замес смеси списывает ингредиенты и добавляет готовую смесь; ежедневный расход смеси считается динамически по назначенным рационам.
- Добавлена команда `/share` и кнопка `Поделиться ботом`; ссылка открывает бота для другого Telegram-аккаунта с отдельным профилем.
- Добавлены тесты изоляции партий, кормов и поголовья по Telegram ID.
- Раздел групп птицы переименован в поголовье; добавлены основное стадо и цыплята.
- Для цыплят добавлен расчет расхода корма по дате вывода с запасом 10% и остановкой отдельного расхода после даты подсадки.
- Добавлены миграции SQLite через `schema_migrations`.
- Добавлены `notification_log`, `analytics_events`, `critical_errors`.
- Добавлены настройки пользователя и пометка неактивных пользователей.
- Добавлены группы птицы, история операций кормов, архивирование кормов и таблицы рецептов.
- Reminder loop пишет журнал уведомлений, не дублирует отправленные события и обрабатывает Telegram blocked/deactivated/network errors.
- Контент инкубации, ухода, disclaimer и рецепт смеси вынесены в JSON с версией.
- Добавлены `/settings`, `/timezone`, `/farm`, `/units`, `/disclaimer`.
- Добавлена `/admin`-статистика, CSV-экспорт и сервисная рассылка с подтверждением.
- Добавлены Docker Compose, systemd-шаблоны, scripts для migrate/backup/restore/check_disk/smoke_start.
- Обновлена документация и тесты.
- Пользовательские `timezone` и `notification_time` стали фактическим источником времени уведомлений.
- Инкубационные уведомления логируются по каждой партии с `batch_id` и event key вида `incubation:batch_ID:day_N:DATE`.
- Добавлены post-hatch-care reminders с дедупликацией через `notification_log`.
- Feed reminders учитывают `is_active` и `notify_feed`.
- Добавлен пользовательский сценарий групп птицы в разделе кормов.
- Добавлен внешний скрипт `notify_admin_failure.py` и systemd `OnFailure` unit.
- Production больше не читает токен из `id/id.txt`.
- Добавлено уведомление пользователей о новой бета-версии после деплоя: сообщение отправляется основным процессом бота при старте, открывает главное меню и дедуплицируется через `notification_log`.
- В расчете кормов добавлены отдельные количества и нормы расхода для кур/несушек и петухов.
